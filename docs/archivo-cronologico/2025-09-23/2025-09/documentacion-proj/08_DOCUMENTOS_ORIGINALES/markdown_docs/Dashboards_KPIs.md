# Dashboards Iniciales (KPIs Básicos)

## Descripción
Este documento describe los dashboards iniciales que se implementarán para el seguimiento de los indicadores clave de rendimiento (KPIs) relacionados con las inscripciones y los pagos. Los dashboards proporcionarán una visión general del estado actual del proyecto y ayudarán en la toma de decisiones.

## KPIs a Incluir
1. **Inscritos Totales**: Número total de inscripciones realizadas.
2. **Pagos vs Pendientes**: Comparativa entre los pagos recibidos y los pagos pendientes.
3. **Fichas Sin Adjunto**: Cantidad de fichas médicas que aún no han sido adjuntadas por los participantes.

## Diseño del Dashboard
- **Tarjetas de KPIs**: Cada KPI se mostrará en una tarjeta que incluirá el valor actual y, si es posible, una comparación con el periodo anterior.
- **Filtros**: Se implementarán filtros para permitir a los usuarios segmentar los datos por fecha, estado de inscripción, y otros criterios relevantes.

## Criterios de Aceptación
- El dashboard debe mostrar datos reales en el entorno de staging.
- Los KPIs deben actualizarse en tiempo real o con la frecuencia definida por el equipo de desarrollo.

## Consideraciones Técnicas
- Utilizar herramientas de visualización de datos que se integren con la base de datos del proyecto.
- Asegurar que el dashboard sea responsivo y accesible desde diferentes dispositivos.
